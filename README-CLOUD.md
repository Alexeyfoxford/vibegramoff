# VibeGram – облачная синхронизация и звонки

По умолчанию VibeGram использует **настоящий SQLite через sql.js (WASM)**, сохранённый в IndexedDB — полноценный SQL с FK, индексами, транзакциями. Но это локально, для одного браузера.

## Подключение Turso (реально работает, не заглушка)

1. Зарегистрируйтесь: https://turso.tech
2. `turso db create vibegram`
3. `turso db tokens create vibegram` — скопируйте токен (это секрет, храните его так же аккуратно, как пароль; если он когда-то "засветился" — перевыпустите новый тем же способом)
4. В приложении: кнопка **☁️ Облако** в верхней панели → вставьте:
   - Turso URL = `libsql://имя-базы.turso.io`
   - Turso Auth Token = `eyJ...`
   - «Сохранить»
5. Готово — таблица создаётся автоматически при первом подключении, дальше все чаты/сообщения/звонки синхронизируются между устройствами и пользователями, у которых указана та же база.

Как это работает: локальная таблица `docs` (общее документное хранилище) копируется в такую же таблицу в Turso. Каждая запись сразу отправляется в облако (push), а раз в ~2.5 секунды опрашиваются чужие изменения (pull) — см. `src/localdb/sync.ts`. Это не настоящий realtime (WebSocket), а polling, но для мессенджера такого масштаба этого достаточно, а зависимостей меньше.

Ограничение: удаления с ДРУГОГО устройства не долетают автоматически (нет tombstone-таблицы) — на практике не критично, т.к. каждый чистит свои чаты сам.

## Звонки — свои, без ZegoCloud

Аудио/видеозвонки 1:1 реализованы через WebRTC напрямую (P2P), а обмен служебными данными для установки соединения (SDP offer/answer, ICE-кандидаты) идёт через ту же Turso-базу — коллекция `calls` в таблице `docs`. Отдельный realtime-сервер для этого не нужен. См. `src/localdb/calls.ts`.

Если звонок не проходит (сложная сеть/NAT, корпоративный файрвол) — добавьте свой TURN-сервер: там же в **☁️ Облако** есть поля TURN URL / логин / пароль (по умолчанию используются только публичные STUN-серверы Google, без TURN-релея).

Групповые звонки в этой версии не поддерживаются (были у ZegoCloud, собственная P2P-реализация первой версии — только 1:1).

## Вариант 2 – Cloudflare D1 (альтернатива Turso)

1. `wrangler d1 create vibegram`
2. `wrangler d1 execute vibegram --file=./public/vibegram-schema.sql`
3. Задеплойте `public/worker-d1.js` как Worker.

Это отдельная, независимая от Turso-синка схема — использовалась раньше для звонков через ZegoCloud (сейчас не нужна) и как альтернативный REST-бэкенд. Для обычного использования сейчас проще и полнее работает вариант с Turso выше.

## Что уже есть локально

- Таблицы: users, chats, messages, bans, reports, contacts, bots, avatar_proposals
- FK, индексы, транзакции
- BanManager, авто-модерация
- E2EE
- Reactions (Twemoji SVG)
- Файлы до 25 МБ с предпросмотром
- Голосовые сообщения, свои видеозвонки WebRTC (см. выше)
- 4 минималистичные темы, видео-обои
- Избранное, история звонков
- Контакты с кастомными именами, предложение аватара
- Группы/каналы публичные/приватные, роли, права
- Бот-конструктор
- Розыгрыши в группах/каналах
- Облачная синхронизация через Turso (см. выше)
- Админ-панель: доступ только через isAdmin в защищённых Firestore Rules (см. SECURITY.md)
