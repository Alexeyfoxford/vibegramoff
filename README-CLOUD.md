# VibeGram – облачная синхронизация и звонки

По умолчанию VibeGram работает **полностью локально** — на настоящем SQLite (sql.js/WASM, тот же движок, что и sqlite3 в Python), сохранённом в IndexedDB. Никакого сетевого запроса, никакого "подождите, идёт синхронизация" — это не заглушка и не урезанная версия, а полноценная встроенная база с транзакциями, индексами и FK.

Единственное, что честно нужно понимать: сама по себе локальная SQLite-база не может передать данные на ДРУГОЕ устройство/другому человеку — у веб-страницы физически нет доступа к чужому компьютеру. Для этого нужен облачный мост, и это ЦЕЛИКОМ опционально.

## Подключение Turso (опционально, для синка между устройствами)

По умолчанию в коде нет никаких зашитых токенов — это open-source-safe: чтобы включить облако, задайте переменные окружения при сборке.

1. Зарегистрируйтесь: https://turso.tech
2. `turso db create vibegram`
3. `turso db tokens create vibegram` — скопируйте токен (это секрет, храните так же аккуратно, как пароль)
4. Скопируйте `.env.example` в `.env` и заполните:
   ```
   VITE_TURSO_URL=libsql://имя-базы.turso.io
   VITE_TURSO_TOKEN=eyJ...
   ```
5. Пересоберите (`npm run build`) — готово, таблица создаётся автоматически при первом подключении.

Как это работает: локальная таблица `docs` (общее документное хранилище) копируется в такую же таблицу в Turso. Каждая запись сразу отправляется в облако (push), а раз в ~2.5 секунды опрашиваются чужие изменения (pull) — см. `src/localdb/sync.ts`. Это не настоящий realtime (WebSocket), а polling.

Ограничение: удаления с ДРУГОГО устройства не долетают автоматически (нет tombstone-таблицы) — на практике не критично, т.к. каждый чистит свои чаты сам. Также у Turso есть бесплатные квоты — это ограничение сервиса, а не кода; если нужен полностью безлимитный вариант без стороннего сервиса — см. вариант с Cloudflare D1/своим сервером ниже, либо разверните собственный libSQL-сервер (sqld) — тогда квот не будет вовсе.

## Звонки — свои, без ZegoCloud

Аудио/видеозвонки 1:1 реализованы через WebRTC напрямую (P2P), а обмен служебными данными для установки соединения (SDP offer/answer, ICE-кандидаты) идёт через ту же Turso-базу — коллекция `calls` в таблице `docs`. Если облако не настроено — звонки внутри одного браузера/устройства всё равно работают (сигналинг идёт через локальную базу), но для звонка МЕЖДУ разными устройствами нужно то же облако, что и для сообщений.

Если звонок не проходит (сложная сеть/NAT, корпоративный файрвол) — можно указать свой TURN-сервер через `setTurnConfig()` (см. `src/localdb/sync.ts`); в интерфейсе поля для этого сейчас нет, только программно.

Групповые звонки в этой версии не поддерживаются (были у ZegoCloud, собственная P2P-реализация первой версии — только 1:1).

## Вариант 2 – Cloudflare D1 (альтернатива Turso)

1. `wrangler d1 create vibegram`
2. `wrangler d1 execute vibegram --file=./public/vibegram-schema.sql`
3. Задеплойте `public/worker-d1.js` как Worker.

Независимая от Turso-синка схема, отдельный REST-бэкенд — на выбор.

## Что уже есть локально

- Таблицы: users, chats, messages, bans, reports, contacts, bots, avatar_proposals, wallposts, friendships, notifications
- FK, индексы, транзакции
- BanManager, авто-модерация
- E2EE в ЛС
- Reactions (Twemoji SVG)
- Файлы с предпросмотром (см. src/localdb/storage.ts про честный лимит размера)
- Голосовые сообщения, свои видеозвонки WebRTC (см. выше)
- 4+ минималистичные темы, видео-обои
- Избранное, история звонков, множественное выделение сообщений
- Стена профиля, друзья (заявки), уведомления — в духе старого ВК
- Группы/каналы публичные/приватные, роли, права
- Бот-конструктор, розыгрыши в группах/каналах
- Облачная синхронизация через Turso — опционально (см. выше)
- Админ-панель: доступ только через isAdmin в защищённых Firestore Rules (см. SECURITY.md)

Всё работает оффлайн, данные кэшируются.
